## Go Maps

- Maps are like superboxes that hold stuff in pairs — a key and a value.

- Every thing in a map has a key (like a name) and a value (like a number or info).

- Maps don’t keep things in order, and you can change them anytime. But you can’t have two of the same key.

- The size of a map is how many key:value pairs it has. You can check it with len().

- If a map is empty, its value is nil therefore the default of map is (nil)

- Maps actually point to a hidden table where all the stuff is stored.

- In Go, you can make maps in different ways.

* One way is by using make()
